# 中文配音替换教程

这份文档用《行军》作为主例子，说明这个工程里语音从脚本到采样文件的查找方式，以及替换中文配音时要注意什么。最后再用《花火》补一个例子，方便以后查其它关卡。

## 基本概念

本工程里的很多“人声”不是一个脚本直接播放 `.wav`，而是：

```text
脚本或引擎调用一个 sfx
-> audio/song_headers.inc.c 里找到 SongHeader
-> SongHeader 指定 MIDI 文件和乐器银行
-> MIDI 里用 Program Change 选择乐器
-> 乐器银行指向某个 sample_xxx.wav
-> 游戏里实际播放这个 sample
```

所以不要只听 `.mid`。普通播放器打开 `.mid` 时，会用电脑自己的 MIDI 音色库播放，听起来通常不是游戏里的人声。真正的人声多数在 `audio/samples/sample_xxx.wav`。

查找时要按这条链路：

```text
audio/sequences/*.mid
-> audio/song_headers.inc.c 的 Bank Number
-> audio/instrument_banks.inc.c 的 bank + Program 槽位
-> audio/instruments/instruments_bankXX.inc.c 的 Sample
-> audio/samples/sample_xxx.wav
```

注意：`instrument_pcm_0109` 里的 `0109` 是乐器结构编号，不等于 `sample_109.wav`。

## 行军例子

行军脚本主要在：

```text
games/marching_orders/subroutines.bs
games/marching_orders/marching_orders.bs
games/marching_orders/marching_orders_2.bs
```

口号 ID 在：

```text
games/marching_orders/macros.inc
```

实际口号音频表在：

```text
games/marching_orders/engine.c
```

关键表是 `marching_sfx_table`。脚本里调用：

```asm
marching_orders_play_voice_sfx VOICE_ATTENTION
```

会进入 `engine.c` 里的 `marching_sfx_table`，再找到对应的 `s_guntai_*_seqData` 或 `s_f_marcher2_*_seqData`。

### 具体追踪示例：`s_guntai_v_zentai1`

1. 在 `games/marching_orders/engine.c` 搜 `s_guntai_v_zentai1_seqData`。
2. 在 `audio/song_headers.inc.c` 搜 `s_guntai_v_zentai1_seqData`。
3. 可以看到它使用：

```c
/* MIDI Sequence */ s_guntai_v_zentai1_mid,
/* Bank Number   */ INST_BANK_4,
```

4. 打开 `audio/sequences/s_guntai_v_zentai1.mid`，看 MIDI 里的 Program Change。当前这个文件使用 `Program 40`。
5. 打开 `audio/instrument_banks.inc.c`，找到 `inst_bank_04` 的 `/* 040 */` 槽位：

```c
/* 040 */ { .pcm = &instrument_pcm_0109 },
```

6. 打开 `audio/instruments/instruments_bank04.inc.c`，找到 `instrument_pcm_0109`，可以看到它实际引用：

```c
/* Sample */ &sample_076_data,
```

7. 所以这句对应的实际采样是：

```text
audio/samples/sample_076.wav
```

如果要替换这句，应优先替换 `sample_076.wav`，不是 `sample_109.wav`。

### 建议中文拆分

行军的日语号令经常不是一句完整中文，而是拆成多个短音频配合节奏。中文也建议拆短。

| 口号 ID | 建议中文 |
| - | - |
| `VOICE_ATTENTION` | 全体！ |
| `VOICE_MARCH` | 前进！ |
| `VOICE_HALT` | 停！ |
| `VOICE_RIGHT_FACE` | 向右！ |
| `VOICE_TURN_RIGHT` | 转！ |
| `VOICE_LEFT_FACE` | 向左！ |
| `VOICE_TURN_LEFT` | 转！ |
| `SHOUT_START` | 嘿！ |
| `SHOUT_STOP` | 哟！ |
| `SHOUT_TURN` | 哈！ |

如果录成“向右转”整句，可能会拖拍。更稳的是“向右！”和“转！”分开替换。

## 替换音频的做法

最稳的入门做法是：找到原采样文件，保留文件名，用新的中文录音覆盖它。

例如行军“全体”：

```text
原文件：audio/samples/sample_076.wav
新文件：仍然命名为 sample_076.wav
```

替换后重新编译：

```bash
make -j
```

如果新录音比原文件长很多，游戏节奏可能不对，ROM 体积也可能变大。建议尽量把中文录音剪到接近原时长。

## 录音规格建议

建议先用这些规则处理新录音：

```text
格式：WAV
声道：单声道
音量：不要爆音，峰值留一点余量
开头：尽量无空白
结尾：尽量干净，不要拖很长尾巴
长度：尽量接近原采样
```

具体采样率可以先参考原文件。替换前用音频软件打开原 `sample_xxx.wav`，新录音尽量导出成同类规格。

如果使用 `ffmpeg`，常见处理思路是：

```bash
ffmpeg -i input.wav -ac 1 output.wav
```

如果需要压短，可以先在 Audacity、Reaper、Adobe Audition 等软件里剪掉空白，再导出。

## 常见坑

1. `.mid` 不是人声本体。很多 `.mid` 只是触发某个 sample。
2. `instrument_pcm_0109` 不等于 `sample_109.wav`。一定要打开乐器定义看 `Sample` 字段。
3. Program 编号要对应 `instrument_banks.inc.c` 里的槽位。比如 `Program 40` 对应 `/* 040 */`。
4. 不要只替换一代。行军 2 用的是 `s_f_marcher2_*` 另一套资源。
5. 不要录太长。节奏游戏最怕提示音拖拍。
6. 替换后声音太大、太小，可以先看 `song_headers.inc.c` 的 `Volume`，也可以看关卡自己的音量表。
7. 如果声音音高不对，可能是 sample_table 的 `pitch`、乐器的 `Key`、或者关卡表里的 `Pitch` 在影响。
8. 如果新声音完全不播，先确认文件名没变、路径没变、WAV 格式能被构建工具处理。

## 第二例子：花火

花火的脚本在：

```text
games/fireworks/fireworks.bs
games/fireworks/macros.inc
games/fireworks/engine.c
```

练习里会直接播放这些语音或提示音：

```asm
play_sfx s_hanabi_1_seqData
play_sfx s_hanabi_2_seqData
play_sfx s_hanabi_3_seqData
play_sfx s_hanabi_hai_seqData
```

`fireworks_play_sfx SFX_NUEI` 这种则会通过 `games/fireworks/engine.c` 的事件播放。

### 花火常见语音

| 用途 | SongHeader | MIDI | 位置 |
| - | - | - | - |
| 1 | `s_hanabi_1_seqData` | `audio/sequences/s_hanabi_1.mid` | `fireworks.bs` |
| 2 | `s_hanabi_2_seqData` | `audio/sequences/s_hanabi_2.mid` | `fireworks.bs` |
| 3 | `s_hanabi_3_seqData` | `audio/sequences/s_hanabi_3.mid` | `fireworks.bs` |
| 嗨 / ハイ | `s_hanabi_hai_seqData` | `audio/sequences/s_hanabi_hai.mid` | `fireworks.bs` |
| 发射喊声 | `s_hanabi_utiage_v_seqData` | `audio/sequences/s_hanabi_utiage_v.mid` | `engine.c` |
| Tamaya | `s_f_hanabi_v_tamaya_seqData` | `audio/sequences/s_f_hanabi_v_tamaya.mid` | 音频资源 |
| Nuei | `s_f_hanabi_v_nuei_seqData` | `audio/sequences/s_f_hanabi_v_nuei.mid` | 音频资源 |

### 具体追踪示例：`s_hanabi_hai`

1. 在 `games/fireworks/fireworks.bs` 搜 `s_hanabi_hai_seqData`。
2. 在 `audio/song_headers.inc.c` 搜 `s_hanabi_hai_seqData`。
3. 可以看到它使用：

```c
/* MIDI Sequence */ s_hanabi_hai_mid,
/* Bank Number   */ INST_BANK_7,
```

4. 打开 `audio/sequences/s_hanabi_hai.mid`，看 Program Change。当前这个文件使用 `Program 11`。
5. 打开 `audio/instrument_banks.inc.c`，找到 `inst_bank_07` 的 `/* 011 */` 槽位。
6. 再打开 `audio/instruments/instruments_bank07.inc.c`，根据该槽位对应的 `instrument_pcm_xxxx` 找到实际 `sample_xxx_data`。
7. 最后替换对应的 `audio/samples/sample_xxx.wav`。

花火这个例子和行军一样：不要直接把中文录音改名成 `.mid`。要替换实际 sample，或者重新设计 MIDI + 乐器银行。

## 推荐工作流

1. 先查清楚 `SongHeader`、`Bank Number`、`Program`、`Sample`。
2. 备份原 `sample_xxx.wav`。
3. 用中文录音覆盖同名 sample。
4. `make -j` 编译。
5. 用 mGBA 测试实际节奏和音量。
6. 太长就剪短，太小声就调音量，节奏不对就重新拆句。

先从一条短口号做测试，比如行军的“全体！”或花火的“嗨！”。确认流程跑通后，再批量替换。
