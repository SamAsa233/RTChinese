# 配字幕方案：以行军为例

这份文档说明如何给《行军》这种原本没有全程字幕的关卡加字幕。重点是先复用工程已有的文字显示系统，避免一开始就改引擎或做新 UI。

## 基本结论

《行军》已经有可用的文字显示通道：

```text
include/beatscript.inc
-> print_text_f
-> common_event 1
-> games/marching_orders/engine.c 的 DISPLAY_TEXT
-> marching_common_display_text
```

也就是说，脚本里可以直接写：

```asm
print_text_f D_0805c9ac
clear_text_f
```

行军脚本里已经用过这种方式，例如 `marching_orders.bs` 里显示“しゅつどう！！”“れんしゅうを　はじめる！”等提示。所以给口号加字幕，优先用 `print_text_f`。

## 相关文件

| 文件 | 用途 |
| - | - |
| `games/marching_orders/marching_orders_text.c` | 放字幕文本 |
| `games/marching_orders/subroutines.bs` | 口号调用最集中的地方 |
| `games/marching_orders/marching_orders.bs` | 一代行军主流程和练习流程 |
| `games/marching_orders/marching_orders_2.bs` | 行军 2 主流程 |
| `games/marching_orders/macros.inc` | 口号 ID、事件宏 |
| `games/marching_orders/engine.c` | 引擎事件表，确认有 `DISPLAY_TEXT` |
| `include/beatscript.inc` | `print_text_f` / `clear_text_f` 宏定义 |

## 字幕文本放哪里

推荐直接加在：

```text
games/marching_orders/marching_orders_text.c
```

例如在文件末尾追加：

```c
const char D_0805c9ac[] = "全体！";
const char D_0805c9b8[] = "前进！";
const char D_0805c9c4[] = "停！";
const char D_0805c9cc[] = "向右！";
const char D_0805c9d8[] = "向左！";
const char D_0805c9e4[] = "转！";
```

注意：这些 `D_0805c9ac` 名字只是反编译风格标签，不一定真是原 ROM 地址。新增时可以沿用类似命名，也可以先用更易读的名字。但如果项目里普遍用 `D_...`，保持风格更省心。

## 最小测试

先不要一口气给所有口号加字幕。建议先选一个最短的口号测试，比如 `VOICE_ATTENTION`。

在 `subroutines.bs` 里找到：

```asm
script marching_orders_sub_08a83758
    marching_orders_event01
    marching_orders_play_voice_sfx VOICE_ATTENTION
    rest 24
    rest 24
return
```

改成类似：

```asm
script marching_orders_sub_08a83758
    marching_orders_event01
    marching_orders_play_voice_sfx VOICE_ATTENTION
    print_text_f D_0805c9ac
    rest 24
    clear_text_f
    rest 24
return
```

这表示口号响起时显示“全体！”，一拍后清掉。

如果你希望字幕停留两拍，可以这样：

```asm
script marching_orders_sub_08a83758
    marching_orders_event01
    marching_orders_play_voice_sfx VOICE_ATTENTION
    print_text_f D_0805c9ac
    rest 24
    rest 24
    clear_text_f
return
```

但字幕停太久会盖住下一句，所以要进游戏实际看。

## 行军口号插入点

`subroutines.bs` 里常见口号：

| 口号 | 插入字幕建议 |
| - | - |
| `VOICE_ATTENTION` | `print_text_f D_0805c9ac`，显示“全体！” |
| `VOICE_MARCH` | `print_text_f D_0805c9b8`，显示“前进！” |
| `VOICE_HALT` | `print_text_f D_0805c9c4`，显示“停！” |
| `VOICE_RIGHT_FACE` | `print_text_f D_0805c9cc`，显示“向右！” |
| `VOICE_RIGHT_FACE_F` | 同上，显示“向右！” |
| `VOICE_TURN_RIGHT` | `print_text_f D_0805c9e4`，显示“转！” |
| `VOICE_LEFT_FACE` | `print_text_f D_0805c9d8`，显示“向左！” |
| `VOICE_LEFT_FACE_F` | 同上，显示“向左！” |
| `VOICE_TURN_LEFT` | `print_text_f D_0805c9e4`，显示“转！” |
| `SHOUT_START` | 可不加，或显示“嘿！” |
| `SHOUT_STOP` | 可不加，或显示“哟！” |
| `SHOUT_TURN` | 可不加，或显示“哈！” |

建议先只给有明确指令意义的几句加字幕：

```text
全体 / 前进 / 停 / 向右 / 向左 / 转
```

纯喊声可以不加，否则画面会很忙。

## 示例：前进

原脚本：

```asm
script marching_orders_sub_08a837ac
    marching_orders_event01
    marching_orders_play_voice_sfx VOICE_MARCH
    rest 24
    marching_orders_play_voice_sfx SHOUT_START
    marching_orders_event00 263
    spawn_cue CUE_STEP
    beat_anim
    rest 24
return
```

加字幕后：

```asm
script marching_orders_sub_08a837ac
    marching_orders_event01
    marching_orders_play_voice_sfx VOICE_MARCH
    print_text_f D_0805c9b8
    rest 24
    clear_text_f
    marching_orders_play_voice_sfx SHOUT_START
    marching_orders_event00 263
    spawn_cue CUE_STEP
    beat_anim
    rest 24
return
```

## 示例：停止

原脚本片段：

```asm
marching_orders_event01
marching_orders_play_voice_sfx VOICE_HALT
marching_orders_event00 8
play_sfx s_guntai_foot1_seqData
spawn_cue CUE_HALT
beat_anim
rest 24
```

可改为：

```asm
marching_orders_event01
marching_orders_play_voice_sfx VOICE_HALT
print_text_f D_0805c9c4
marching_orders_event00 8
play_sfx s_guntai_foot1_seqData
spawn_cue CUE_HALT
beat_anim
rest 24
clear_text_f
```

这里清字幕放在 `rest 24` 后面，是为了让“停！”显示完整一拍。

## 示例：向右转

原脚本片段：

```asm
marching_orders_event01
marching_orders_play_voice_sfx VOICE_RIGHT_FACE
rest 24
rest 24
marching_orders_event01
marching_orders_play_voice_sfx VOICE_TURN_RIGHT
spawn_cue CUE_TURN_RIGHT
beat_anim
rest 24
marching_orders_event00 4
marching_orders_play_voice_sfx SHOUT_TURN
rest 24
```

可改为：

```asm
marching_orders_event01
marching_orders_play_voice_sfx VOICE_RIGHT_FACE
print_text_f D_0805c9cc
rest 24
rest 24
marching_orders_event01
marching_orders_play_voice_sfx VOICE_TURN_RIGHT
print_text_f D_0805c9e4
spawn_cue CUE_TURN_RIGHT
beat_anim
rest 24
clear_text_f
marching_orders_event00 4
marching_orders_play_voice_sfx SHOUT_TURN
rest 24
```

这样第一段显示“向右！”，真正转向时显示“转！”。

## 编码和字体注意

这是最大坑。

如果直接在 `.c` 里写：

```c
const char D_0805c9ac[] = "全体！";
```

不代表游戏一定能显示。要看当前工程的文字系统是否支持中文字符。常见情况：

1. 编译能过，但游戏里乱码。
2. 游戏里空白，因为字库没有这个字。
3. 宽度不对，字挤在一起或超出气泡。
4. 文件编码被编辑器改坏，原日文也变成乱码。

所以建议先做最小测试：

```c
const char D_0805c9ac[] = "全体！";
```

只在一个地方 `print_text_f D_0805c9ac`，编译进游戏看。如果能显示，再批量加。

如果不能显示，有两条路：

1. 修中文字库和编码。
2. 把字幕做成图片或 tilemap，不走文字系统。

新手优先试文字系统，因为行军已经有 `print_text_f`。如果字库不支持中文，再考虑图片字幕。

## 字幕时长建议

行军节奏里 `rest 24` 通常是一拍。字幕显示建议：

```text
短口号：显示 1 拍
预备口号：显示 1 到 2 拍
连续口号：下一句出现时直接覆盖上一句
关卡结束或段落结束：记得 clear_text_f
```

不要每个脚步声都加字幕，只给口令加。字幕太密会干扰玩家看输入。

## 推荐实施顺序

1. 在 `marching_orders_text.c` 加一条测试字幕，比如“全体！”。
2. 在 `marching_orders_sub_08a83758` 里加 `print_text_f`。
3. 编译并进游戏测试中文是否显示。
4. 如果中文显示正常，再加“前进！”和“停！”。
5. 最后再加“向右！”“向左！”“转！”。
6. 测试练习、正式关、行军 2，确认没有字幕残留。

## 字幕残留排查

如果某句字幕一直留在屏幕上，通常是漏了：

```asm
clear_text_f
```

可以在每个口号段落结束后清一次，也可以在下一句字幕出现时直接覆盖。遇到流程分支或关卡结束前，最好主动清掉。

## 是否需要改引擎

第一阶段不需要。

只有遇到这些情况，才考虑改引擎：

1. `print_text_f` 的位置不合适，挡住玩法。
2. 想做固定底部字幕条。
3. 想同时显示两行或多行字幕。
4. 中文字库无法用现有文字系统承载。

在那之前，直接改 `.c` 文本和 `.bs` 脚本就够了。
